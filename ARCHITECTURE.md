## Key Architectural Decisions

### 1. Why VITE_ Prefix?
- **Security**: Vite only exposes prefixed vars to client
- **Explicit**: Clear which vars are client-accessible
- **Standard**: Follows Vite best practices

### 2. Why Multiple API Keys?
- **Reliability**: Automatic failover on rate limits
- **Capacity**: 3x the daily quota
- **Resilience**: Service continues if one key fails

### 3. Why Caching?
- **Performance**: 90% reduction in API calls
- **Cost**: Stay within free tier limits
- **UX**: Faster page loads

### 4. Why Server-Side NewsAPI?
- **CORS**: Avoid browser restrictions
- **Security**: Centralized key management
- **Monitoring**: Better error tracking

### 5. Why Client-Side GNews/YouTube?
- **Simplicity**: Direct API calls
- **Performance**: No server round-trip
- **Scalability**: Offload from server
